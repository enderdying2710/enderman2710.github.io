校鱼校园二手交易平台 — 后端安全与性能升级日志

项目：校鱼 (XiaoYu) · uni-app + Spring Boot 2.7 + MongoDB
日期：2026-05-19
分支：appmod/java-upgrade-20260519012250

📋 概览

本次升级围绕安全性、性能、实时体验三个维度对后端和前端进行了系统优化。涉及文件 57 个，新增 450 行，删除 308 行。

一、🔐 JWT 认证体系搭建（安全升级）

变更前

旧系统采用简陋的 X-User-Id 请求头认证 —— 前端从 uni.storage 读取 userInfo.id 直接拼在请求头上，无校验、无过期、无法防范伪造。

变更后

引入 JWT (JSON Web Token) 认证机制，包含三个核心组件：

组件 路径 职责
JwtUtil config/JwtUtil.java Token 生成、解析、校验
AuthInterceptor config/AuthInterceptor.java 拦截 /api/** 请求，验证 Authorization: Bearer <token>
WebConfig config/WebConfig.java 注册拦截器，白名单配置

认证流程：

1. 用户登录 → 后端返回 { user, token }
2. 前端存储 token → 每次请求携带 Authorization: Bearer <token>
3. AuthInterceptor 拦截 → 校验 token → 提取 userId → 放行
4. 未登录接口（验证码/登录/上传/静态资源）→ 白名单跳过

白名单接口： /api/user/sendCode、/api/user/login、/api/user/loginByPassword、/api/upload/**、/images/**

Token 有效期： 7 天（604800000ms），可通过 JWT_EXPIRATION 环境变量配置

token 保密等级： JWT Secret 通过 ${JWT_SECRET:...} 环境变量注入，默认值为占位字符串

二、🔒 BCrypt 密码哈希（安全升级）

变更前 

// 明文存储 + 明文比较（严重安全隐患）
user.setPassword(password);  // 原始密码直接存库
if (!user.getPassword().equals(password)) ...  // 字符串直接比较

变更后 

// BCrypt 加盐哈希
user.setPassword(passwordEncoder.encode(password));  // $2a$10$... 密文
if (!passwordEncoder.matches(password, user.getPassword())) ...  // BCrypt 匹配

新增依赖：spring-security-crypto（已内置于 Spring Security，无版本冲突风险）

密码接口增强：

  • 密码未设置时提示「请先使用验证码登录后设置密码」
  • getUser 接口返回前自动清除 password 字段，杜绝密码泄露

三、📄 分页查询（性能优化）

问题

列表接口（商品/需求）一次性返回全部数据，数据量增长时响应变慢、前端渲染压力大。

改动

前后端同步支持分页，契约约定 page 从 0 开始，size 默认 20。

后端：

// ProductController · DemandController
@GetMapping("/list")
public ApiResult<List<Product>> list(
    @RequestParam(defaultValue = "0") int page,
    @RequestParam(defaultValue = "20") int size
) {
    return ApiResult.ok(productService.findAllPaged(page, size));
}

// ProductService · DemandService
public List<Product> findAllPaged(int page, int size) {
    return repository.findByStatusOrderByCreateTimeDesc("online",
            PageRequest.of(page, size));
}

Repository 新增支持 Pageable 的重载方法。

前端：

// utils/api.js
list(page = 0, size = 20) {
    return request(`/api/product/list?page=${page}&size=${size}`)
}

// goods.vue - 触底加载更多
const loadMore = () => {
    if (loading.value || noMore.value) return
    goodsPage.value++
    loadGoods(false)  // false = 追加模式，不清空列表
}

SPA 式分页交互：下拉触底 → 加载下一页 → 追加到列表 → 数据不足一页时显示「没有更多了」。

四、💬 聊天消息轮询（实时体验）

问题

聊天页面只加载一次消息，用户需要手动刷新或退出重进才能看到新消息。

改动

chat.vue 新增 3 秒轮询机制：

// 启动轮询（每3秒检查新消息）
const startPolling = () => {
    pollTimer = setInterval(async () => {
        const msgs = await chatApi.messages(chatId.value)
        if (msgs && msgs.length > lastMsgCount) {
            // 发现新消息 → 全量刷新列表 → 标记已读 → 滚动到底部
            messageList.value = /* ...映射... */
            lastMsgCount = msgs.length
            chatApi.markRead(chatId.value, myUserId.value)
        }
    }, 3000)
}

配套优化：

  • 页面卸载时 onUnmounted 自动停止轮询，避免内存泄漏
  • 后端 ChatMessageRepository 新增 MongoDB 批量标记已读方法：
@Query("{ 'sessionId' : ?0, 'senderId' : { '$ne' : ?1 }, 'isRead' : false }")
@Update("{ '$set' : { 'isRead' : true } }")
void markMessagesAsRead(String sessionId, String userId);

用单条 MongoDB 更新语句替代之前的逐条加载+逐条保存，大幅减少数据库往返。

五、🛡️ 文件上传安全校验（安全加固）

变更前

文件上传接口无任何校验，任意文件类型/大小均可上传。

变更后 

// UploadController
private static final Set<String> ALLOWED_EXTENSIONS = Set.of(
    "jpg", "jpeg", "png", "gif", "webp", "bmp"
);
private static final long MAX_FILE_SIZE = 5 * 1024 * 1024; // 5MB

@PostMapping("/image")
public ApiResult<Map<String, String>> uploadImage(@RequestParam("file") MultipartFile file) {
    if (file.isEmpty()) return ApiResult.fail("文件为空");
    if (file.getSize() > MAX_FILE_SIZE) return ApiResult.fail("文件大小不能超过5MB");
    // 扩展名白名单校验
    if (!ALLOWED_EXTENSIONS.contains(ext)) return ApiResult.fail("不支持的文件类型");
    ...
}

前端同步增加 token 认证和 30s 超时设置：

// 上传也携带 JWT
const token = getToken()
const header = token ? { 'Authorization': 'Bearer ' + token } : {}
uni.uploadFile({ ..., header, timeout: 30000 })

六、🗝️ OSS 密钥环境变量化（安全合规）

改动

阿里云 OSS 配置从硬编码改为环境变量注入：

# 变更前 —— 硬编码（已提交到 Git）
aliyun:
  oss:
    access-key-id: xxxx    # ⚠️ 泄露风险
    access-key-secret: xxxx

# 变更后 —— 环境变量（默认值留空）
aliyun:
  oss:
    access-key-id: ${OSS_ACCESS_KEY_ID:}
    access-key-secret: ${OSS_ACCESS_KEY_SECRET:}

⚠️ 如果已泄露的旧密钥仍在阿里云控制台有效，建议立即删除并重新生成。

七、🧹 代码清理

  • 移除所有 Controller 上的 @CrossOrigin(origins = "*") 注解 —— CORS 已统一由 WebConfig.addCorsMappings 管理
  • ProductService.incViewCount 改用 MongoDB 原子更新（$inc），替代 find → 修改 → save 三步走
  • 前端提取公共常量 CATEGORY_LIST、PAGE_SIZE 到 utils/constants.js
  • 统一 401 过期提示：「登录已过期，请重新登录」
  • 移除不必要的 console.error 调用

📦 新增依赖

依赖 版本 用途
io.jsonwebtoken:jjwt-api 0.11.5 JWT 生成/解析
io.jsonwebtoken:jjwt-impl 0.11.5 JWT 实现（runtime）
io.jsonwebtoken:jjwt-jackson 0.11.5 JSON 序列化（runtime）
spring-security-crypto (继承自 parent) BCrypt 密码哈希
spring-boot-starter-validation (继承自 parent) Bean 校验

⚠️ 注意事项

  1. JWT Secret 生产环境请设置强随机字符串，可通过 JWT_SECRET 环境变量覆盖
  2. OSS 密钥 需要开发者在本地或服务器配置 OSS_ACCESS_KEY_ID / OSS_ACCESS_KEY_SECRET 环境变量
  3. target/ 目录 已从 Git 跟踪中移除 — 下次 mvn clean compile 后重新生成
  4. 聊天轮询目前每 3 秒请求一次，用户量大后可酌情调整为 5 秒或改用 WebSocket
