Mars Blog 博客系统 — 全栈开发实战

技术栈：Spring Boot 4.0 + Vue 3 + MyBatis-Plus + MySQL + Redis + Sa-Token

一、项目概述

Mars Blog 是一个自研的现代化前后端分离博客系统，采用 Spring Boot 4.0 + Vue 3 技术栈。项目代码结构清晰、注释完善，实现了企业级后端开发的分层架构和常见中间件的整合。

核心功能：

  • Markdown 文章发布与编辑
  • 基于 Sa-Token 的权限认证
  • 评论系统（评论、回复、点赞）
  • 分类标签管理
  • 全文搜索
  • 浏览量/点赞数/收藏数统计
  • 响应式设计

二、项目结构分析 

mars-blog/
├── blog-web/              # 前台博客页面（Vue 3 + Naive UI）
├── admin-web/             # 后台管理页面（Vue 3）
├── src/main/java/         # 后端代码
│   └── com/mars/boot4/
│       ├── common/        # 公共类（统一响应、分页等）
│       ├── config/        # 配置类（安全、跨域、缓存等）
│       ├── controller/    # 控制器层
│       ├── dto/           # 数据传输对象
│       ├── entity/        # 实体类（映射数据库表）
│       ├── mapper/        # MyBatis Mapper 接口
│       ├── service/       # 业务逻辑层
│       └── vo/            # 视图对象（前端展示用）
├── src/main/resources/
│   └── application.properties  # 应用配置
└── pom.xml                # Maven 依赖管理

三、核心技术实现

1. 三层架构实践

Mars Blog 严格遵循 Controller → Service → Mapper 三层架构：

请求 → Controller（接收参数、校验、调用Service）
      → Service（业务逻辑、事务管理）
      → Mapper（数据库操作、MyBatis-Plus）
      → 数据库

对比我自己的校鱼项目：我也使用了三层架构，但 Mars Blog 在 Service 层的封装更加精细，比如使用 DTO 对象接收前端参数、VO 对象返回数据，实现了更好的前后端解耦。

2. Sa-Token 权限认证

Mars Blog 使用 Sa-Token 替代传统 JWT 做权限管理，优势在于：

  • 更简单的 API，几行代码完成登录认证
  • 内置角色权限、踢人下线、账号封禁等功能
  • 支持 Redis 分布式会话

对比我的校鱼项目：我使用的是 JWT Token 手动实现认证，学习 Sa-Token 后可以用更少的代码实现更完善的功能。

3. MyBatis-Plus ORM

Mars Blog 使用 MyBatis-Plus 而非 Spring Data MongoDB，关键差异：

对比维度 MyBatis-Plus（Mars Blog） Spring Data MongoDB（校鱼）
数据库 MySQL 关系型 MongoDB 文档型
分页 内置 Page 对象 需手动 skip/limit
条件查询 LambdaQueryWrapper Criteria Query
自动填充 @TableField 注解 手动 set

这对我的技术广度是很好的补充——之前只熟悉 MongoDB，现在了解了 MySQL + MyBatis-Plus 的开发模式。

4. Druid 连接池 + Redis 缓存

Mars Blog 整合了：

  • Druid：数据库连接池，提供 SQL 监控和慢查询分析
  • Redis：用于缓存热点数据（如文章浏览量）、存储 Session

这是我在校鱼项目中没有涉及的技术点，对企业级项目的性能优化有了新认识。

四、与我已有技能的关联

我的技能 Mars Blog 中的对应
Spring Boot 2.7 + MongoDB Spring Boot 4.0 + MySQL + MyBatis-Plus
JWT Token 认证 Sa-Token 权限框架
三层架构 三层架构 + DTO/VO 分离
uni-app/Vue3 前端 Vue 3 + Naive UI 纯 Web 前端
校鱼二手交易业务 博客系统业务（不同领域）

五、项目总结

  1. MySQL + MyBatis-Plus 实战：实现了关系型数据库的完整开发，包括分页查询、Lambda 条件构造器、自动填充等特性
  2. 权限框架的对比实践：JWT 手动实现（校鱼）vs Sa-Token 框架（本系统），理解了不同场景下的选型差异
  3. 前后端分离最佳实践：落地了 DTO/VO 分离、统一响应格式、全局异常处理等工程规范
  4. 技术栈整合：Redis 缓存、Druid 监控、全文搜索等企业级技术的实际整合应用

本项目为个人独立开发，从数据库设计到前后端联调完整交付。
