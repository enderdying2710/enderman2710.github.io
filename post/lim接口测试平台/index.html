Lim 接口测试平台 — 轻量级接口自动化测试

技术栈：React + Python Django + SQLite

一、项目概述

Lim 是 “Less is More”（少即是多）的缩写，一个自研的轻量级接口测试平台。设计理念是让接口测试变得简单且高效——去掉了传统测试平台中很多复杂的概念（如局部变量、前后置计划等），通过创新的交互设计让用户操作更少但效率更高。

核心亮点：

  • 免登录设计：非敏感操作无需登录即可查看报告
  • 参数自动回填：编写过的接口下次自动回填参数
  • Python 代码支持：变量生成、断言判断完全兼容 Python
  • 灵活的循环控制：支持多层嵌套、变量控制循环次数
  • 多环境管理：一套用例可在多套环境下执行

二、项目结构分析 

Lim接口测试平台/
├── LimApi/          # 后端 Django 项目
├── lim-web/         # 前端 React 项目
├── init-db.sql      # 数据库初始化脚本
└── README.md

后端架构（LimApi）

后端基于 Python Django 框架，主要模块：

  • API 管理：接口的增删改查、参数解析
  • 用例管理：测试用例的编排与执行
  • 环境管理：多套环境参数配置
  • 报告生成：测试执行结果汇总

前端架构（lim-web）

前端基于 React，交互亮点：

  • 弹窗可拖拽，灵活查看内容
  • 步骤支持合并、复制、拖拽排序
  • 参数类型自动识别、自动回填

三、核心技术实现

1. 接口参数自动解析

Lim 会自动解析 URL 上的参数并回填到 query 中。这涉及到：

  • URL 解析与参数提取
  • 参数类型推断（String/Number/Boolean）
  • 接口历史记录的匹配与回填

2. Python 代码集成

平台内嵌 Python 执行环境，用于：

  • 动态生成测试变量（如随机手机号、时间戳）
  • 自定义断言逻辑（不只是简单的等值判断）
  • 执行条件控制（if/else 逻辑）
# 示例：生成随机手机号
import random
phone = "138" + "".join([str(random.randint(0, 9)) for _ in range(8)])

3. 多环境切换

通过配置多套环境参数（开发/测试/生产），同一套用例可以快速切换执行环境，核心是变量替换和环境隔离。

4. 步骤控制器

每个测试步骤支持：

  • 延时等待：接口调用间隔控制
  • 重试机制：失败自动重试
  • 执行条件：按条件决定是否执行

四、与我已有技能的关联

我的技能 Lim 中的对应
Apifox/Postman 接口测试 Lim 的接口管理与测试执行
Python 基础 Lim 的 Python 代码集成
测试用例设计 Lim 的用例编排与步骤控制
WPS 表格管理测试数据 Lim 的参数池与环境变量管理

五、项目总结

  1. 接口测试平台的核心功能模块：实现了 API 管理、用例编排、环境切换、报告生成这四大模块
  2. Python 在测试中的实际应用：Python 在测试变量生成、自定义断言中发挥关键作用
  3. 交互设计对效率的影响：免登录、参数自动回填、弹窗拖拽等设计能大幅提升测试效率
  4. 轻量化的价值：不需要大而全，聚焦核心流程并做到极致，比功能堆砌更有价值

本项目为个人独立开发，从需求分析到前后端实现完整交付。
