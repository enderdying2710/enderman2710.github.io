Hugo 博客标签与配置文件修改指南

一、给文章添加标签和分类

在 Hugo 中，标签 (tags) 和分类 (categories) 写在每篇文章的 front matter（文件头部的 --- 区域）里。

基本语法 

---
title: '你的文章标题'
date: '2026-05-21'
tags: ['标签1', '标签2', '标签3']
categories: ['分类名']
---

标签 vs 分类

分类 (categories) 标签 (tags)
性质 大类，一篇文章通常一个分类 细粒度关键词，可以有多个
举例 技术、生活、项目 Hugo、Python、前端、教程
页面 /categories/技术/ /tags/Hugo/

实际例子 

---
title: '我的技术文章'
date: '2026-05-21'
tags: ['Python', 'Django', '后端']
categories: ['技术']
---

添加后 Hugo 会自动生成：

  • /categories/ — 所有分类汇总
  • /categories/技术/ — “技术” 分类下的文章列表
  • /tags/ — 所有标签汇总
  • /tags/Python/ — 含 “Python” 标签的文章列表

二、修改 Hugo 配置文件

站点配置 — hugo.toml

hugo.toml 是项目的根配置文件，位于项目根目录。主要配置项：

baseURL = 'https://你的域名.github.io/'
title = '你的站点标题'
theme = 'hugo-theme-reimu'
defaultContentLanguage = 'zh-cn'

多语言配置：

[languages.en]
label = 'English'
weight = 2

[languages.zh-cn]
label = '简体中文'
weight = 1
hasCJKLanguage = true

主题参数 — themes/hugo-theme-reimu/config/_default/params.yml

这个文件控制主题的所有功能，常用的修改项：

网站信息 

author: 你的名字
description: "不畏苦暗！！"    # 网站描述，会出现在 meta 标签和侧边栏
subtitle:
  text: "不畏苦暗！！"        # 首页副标题

加载动画 (Preloader) 

preloader:
  enable: true
  text:
    zh-CN: 不畏苦暗！！
    en: Loading...

打字机效果 

subtitle:
  typing:
    enable: false           # 设为 true 开启打字动画
    strings:                # 循环播放的句子
      - 第一句话
      - 第二句话
    typeSpeed: 100          # 打字速度（毫秒/字符）
    loop: true              # 是否循环

侧边栏小部件 

widgets:
  - category       # 分类
  - tag            # 标签
  - tagcloud       # 标签云
  - recent_posts   # 最新文章

社交链接 

social:
  github: https://github.com/你的用户名
  bilibili: https://space.bilibili.com/你的ID

特效开关 

firework:
  enable: true              # 鼠标烟花特效

animation:
  enable: true              # 页面滚动动画

reimu_cursor:
  enable: true              # 自定义光标

pace:
  enable: true              # 页面加载进度条

三、修改后如何生效

在项目根目录下运行：

./hugo.exe
# 或者
hugo

Hugo 会重新生成 public/ 目录下的所有静态文件。

如果需要本地预览：

hugo server
# 然后访问 http://localhost:1313

总结

  1. 加标签/分类 → 修改文章 front matter 的 tags 和 categories
  2. 改网站信息 → 修改 params.yml 的 author、description、subtitle
  3. 改功能开关 → 修改 params.yml 对应的 enable
  4. 改多语言 → 修改 hugo.toml 的 [languages]
  5. 每次修改后 → 运行 hugo 重新构建
