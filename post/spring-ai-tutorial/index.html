Spring AI Tutorial — Java AI 开发实战

技术栈：JDK21 + Spring Boot 3.4.5 + Spring AI 1.0.0 + Spring AI Alibaba 1.0.0

一、项目概述

Spring AI Tutorial 是一个系统的 Spring AI 实战项目，涵盖从基础对话到多智能体协作的完整开发路径。项目采用模块化结构，每个功能模块独立为一个 Maven 子工程，可逐个运行和调试。

13 个模块覆盖的核心能力：

  • Chat（对话）：阿里百炼、OpenAI、DeepSeek 多模型接入
  • Advisor（对话记忆）：内存/SQLite/MySQL/Redis 持久化
  • Tool Calling（函数调用）：让 AI 调用 Java 方法
  • RAG（检索增强生成）：向量数据库 + 知识库问答
  • MCP（模型上下文协议）：stdio/WebFlux/Nacos 多种传输
  • Graph/Agent（多智能体）：并行节点、人工介入、流式输出

三、核心技术实现

1. ChatClient 统一对话入口

Spring AI 1.0 引入 ChatClient，以流式 API 风格完成与大模型的对话：

@RestController
public class ChatController {
    private final ChatClient chatClient;

    public ChatController(ChatClient.Builder builder) {
        this.chatClient = builder
            .defaultSystem("你是一个博学的智能聊天助手")
            .build();
    }

    @GetMapping("/chat")
    public String chat(@RequestParam String query) {
        return chatClient.prompt(query).call().content();
    }
}

对比传统开发：以前调用 AI 需要手动拼 HTTP 请求、处理 Token 刷新，现在 Spring AI 通过 ChatClient 将这些细节全部封装，和写普通 Service 一样简单。

2. RAG 检索增强生成

RAG（Retrieval-Augmented Generation）让大模型能基于私有知识库回答问题，核心流程：

文档 → 向量化(Embedding) → 存入向量数据库
                          ↓
用户提问 → 向量化 → 相似检索 → 拼接上下文 → 发给大模型 → 生成回答

项目中 rag-simple 模块展示了有无 RAG 的回答质量对比，让人直观理解 RAG 的价值——没有 RAG 时大模型只能"编造"，有了 RAG 后能基于真实文档给出准确回答。

3. Function Calling / Tool Calling

这是让 AI 与业务系统联动的关键能力。AI 不是直接回答，而是调用你写的 Java 方法来获取实时数据：

用户："今天珠海天气怎么样？"
  ↓
AI 识别意图 → 调用 getWeather("珠海") → 获取实时天气数据
  ↓
AI 将数据组织成自然语言 → "珠海今天多云，25-30°C..."

4. Advisor 对话记忆

让 AI 记住之前的对话上下文，支持多种存储后端：

存储方式 适用场景
内存 开发调试
SQLite 单机部署
MySQL 生产环境持久化
Redis 高性能多轮对话

五、项目总结

  1. Java 生态的 AI 开发路径：Spring AI 让 Java 开发者用熟悉的 Spring Boot 方式开发 AI 应用，降低了 AI 开发的门槛
  2. 大模型应用的核心概念落地：Chat、RAG、Tool Calling、MCP、Agent 这些概念通过实际编码真正落地
  3. 私域知识库的完整实现：RAG 从文档加载到向量检索到答案生成的完整链路，对 AI 测试岗位理解大模型版本更新后的测试重点很有帮助
  4. 模型评测的工程实践：rag-evaluation 模块用代码批量评估不同模型的回答质量，和 WPS AI 测试岗位的评估工作直接相关

本项目为个人独立开发，基于 Spring AI 1.0.0 官方文档与动手实践。
